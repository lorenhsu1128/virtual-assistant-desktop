剛才在這次對話中，你（Claude Code）犯了一個錯誤並被使用者修正。現在需要將這個錯誤記錄到 `LESSONS.md` 中，確保未來的對話不會再犯同樣的錯。

## 步驟

### 1. 回顧錯誤

回顧這次對話中被修正的錯誤：
- 你做了什麼？（具體的錯誤操作）
- 應該怎麼做？（正確做法）
- 哪些檔案受影響？
- 為什麼會犯這個錯？（根本原因分析）

如果使用者在 $ARGUMENTS 中提供了描述，以此為基礎分析。
如果沒有提供，回顧最近的對話內容找出被修正的錯誤。

### 2. 分類

判斷錯誤屬於哪個分類：
- **架構違規**：違反三層架構、模組邊界
- **IPC 通訊**：IPC 介面使用錯誤
- **Rust 後端**：Rust 程式碼錯誤
- **效能問題**：違反效能預算
- **型別問題**：型別定義不一致
- 如果不屬於任何分類，建立新分類

### 3. 寫入 LESSONS.md

在對應分類下新增條目：

```markdown
### [YYYY-MM-DD] — 一句話描述

- **錯誤**：{具體描述做了什麼}
- **正確做法**：{應該怎麼做}
- **受影響檔案**：{檔案路徑}
- **根因**：{為什麼會犯這個錯，幫助 AI 理解邏輯}
```

### 4. 檢查是否需要加強規則

評估這個錯誤是否反映了 CLAUDE.md 或 Skills 中規則不夠明確：

- 如果 CLAUDE.md 中已有規則但被忽略 → 在 LESSONS.md 的記錄中標註「規則已存在但被忽略」
- 如果 CLAUDE.md 中沒有對應規則 → **詢問使用者是否要在 CLAUDE.md 的禁止清單中新增規則**
- 如果是某個 Skill 的步驟遺漏 → 詢問使用者是否要更新 Skill

### 5. 確認

向使用者確認記錄內容，並報告：
- 已寫入 LESSONS.md 的哪個分類
- 是否建議更新 CLAUDE.md 或 Skills
